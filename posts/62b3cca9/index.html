数据结构——图

本文最后更新于：Saturday, August 22nd 2020, 3:03 pm

1、基本概念

  • 多重图：有重复的边

  • 度(degree)：该vertex连接的边数

    针对有向图来说：

    • 出度：以该vertex为始边的个数
    • 入度：以该vertex为终边的个数

  • 路径长度：路径上边的个数

    • 简单路径：路径除了起点和终点相同外，再也不能有相同的顶点。
    • 简单回路：如果简单路径起点和终点相等，且路径长度大于等于2。

  • 子图：设G，H是图，如果$V(H)\subset V(G)$,$E(H)\subset E(G)$,则H是G的子图。G是H的母图

    • 支撑子图：如果H是G的子图，并且V(H) = V(G)。（顶点相同)

  • 连通图：G是无向图，且V(G)中任意两顶点都可及。

    • 强连通图: G是有向图，且任意两个不同的顶点互相可及
    • 连通子图: G是无向图，若G的子图$G_k$是一个连通图。
    • 强连通子图：G是有向图，若G的子图$G_k$是一个强连通图。

  • 连通分量: 对于无向图G的一个连通子图$G_k$，如果不存在G的另一个连通子图$G^{‘}$，使得$V(G_k)\subset V(G^{‘})$, 则称$G_k$是G的连通分量。（结点数最多的连通子图, $G_k$再加一个顶点就不连通了)

    • 强连通分量: 同上，不过是有向图，强连通子图。

2、图的存储结构

1、邻接矩阵

顶点v0,v1,…,$v_{n-1}$用顺序表或链表存储。图用n*n的矩阵表示。

  • 无向图矩阵对称，可以压缩存储空间。有向图不行。$n^2$
  • 无向图：邻接矩阵第i行或第i列非零元素个数是顶点$V_i$的度
  • 有向图：邻接矩阵第i行非零元素个数表示出度，第i列表示入度

2、邻接表

每个顶点建立一个单链表

  • 顺序存储的顶点表和链接存储的边链表——>邻接表

  • 顶点表中结点的结构

    struct Vertex
{
	int VerName;		//顶点名称
	Edge *adjacent;		//边链表的头指针
};

  • 边结点的结构

    struct Edge
{
	int VerAdj;	//邻接顶点
	int cost;	//边的权值
	Edge *link; //指向下一个边结点
};

  • 求入度(时间复杂度O(n+e))

    Algorithm InDegree(Head,n.count) //head为结点数组，n为结点个数，count为入度数组)
For i=0 to n-1 Do
	count[i]<——0.
For i=0 to n-1 Do
( p<——adjacent(Head[i]).
  While p!=^ Do  //应该用数学里面的不等于
  (	k<——VerAdj(p).
    count[k]<——count[k]+1.
    p<——link(p).
  )
)|

for(p<——adjacent(Head[i]); p!=^; p<——link(p))
    ( k<——VerAdj(p).
      count[k]<——count[k]+1.
	)

    建立有向图逆邻接表

  • 顶点多边上的图适合邻接表。（稀疏图n*n)

3、图的遍历

从图的顶点出发，访问图中所有的顶点，且使每个顶点恰被访问一次的过程被称为图的遍历

1、深度优先遍历(Depth First Search)

回溯：可行则进，不进则换，换不了则退

  • 递归实现
